1. Executive Summary
Aether AI is a multi-tenant Agentic AI Chatbot SaaS that lets any business spin up a premium, brand-customised chatbot in minutes. The bot is not a thin wrapper around an LLM — it is an agent with pluggable skills, payments, e-commerce, appointment booking, and webhook automations through n8n. The platform is designed to look and feel like a flagship 2026 consumer product (Linear, Vercel, Arc, Raycast tier polish) while remaining buildable on free-tier infrastructure (Supabase + Vercel/Netlify) and ready to scale to enterprise without a rewrite.
Three surfaces ship together: a marketing landing page, a Super-Admin SaaS console (you control providers, plans, credits, analytics, payments), and a Client/Company dashboard (your subscribers build, customise, and deploy their bots). The chat experience itself is a native-mobile-grade, full-width UI with five premium design presets, deep brand customisation, and rich Markdown / formatting output.
Differentiators: agentic skills marketplace, in-chat commerce (Razorpay + Stripe), WordPress plugin with one-click auth, custom-domain bot hosting, dual-region payments (Stripe global + Razorpay India), and a credit-based metering model that works across every major LLM provider.

4.2 Multi-Tenancy Model
Single Postgres database. Every tenant-owned row carries tenant_id (UUID).
Row-Level Security policies enforce tenant_id = auth.jwt() ->> 'tenant_id' on every table.
Super-Admin role bypasses RLS via a server-only service-role key (never exposed to the browser).
Bots are addressable by slug (aether.ai/b/<slug>), subdomain (<slug>.bots.aether.ai), or custom domain.
4.3 Request Lifecycle (Chat Message)
End-user types a message in the chat widget.
POST /api/chat/[botId] streams to Edge Runtime.
Middleware resolves tenant → loads bot config, theme, skills, model, credit pool.
Credit guard: check tenant balance + per-bot quota in Redis (atomic decrement).
AI SDK builds the prompt: system prompt + RAG context (pgvector) + skill tool schemas.
LLM call (provider chosen per bot: client API key or admin pool key).
If the model returns a tool call → execute skill (built-in handler or n8n webhook).
Stream tokens back to UI; persist message + usage rows; settle credits.
Realtime broadcast → Client Dashboard sees the conversation live.

6.5 Skills System (Agentic Tools)
Skills are the agent's hands. Three flavours:
Built-in skills (we ship them): search_products, create_order, take_payment, list_slots, book_appointment, capture_lead, handoff_to_human, search_knowledge_base, send_email.
Webhook skills (no-code via n8n): client gives a name, description, JSON-schema for arguments, and an n8n webhook URL. The agent learns to call it.
HTTP / SQL skills (advanced): for power users — paste an OpenAPI snippet or a parameterised SQL query.
Skill execution is sandboxed, timeouts enforced, every run logged to skill_runs with input/output for debugging. The bot trainer UI lets clients add few-shot examples so the agent learns when to call each skill.

8.2 Credit Metering
1 credit ≈ 1,000 tokens of gpt-4o-mini-equivalent. Each model has a multiplier (e.g., Claude Sonnet = 3x, Groq Llama = 0.3x).
Decrement atomically in Redis on every successful response; settle to credit_ledger asynchronously.
Bring-your-own-key bypasses metering entirely (client pays the provider directly).
8.3 Razorpay (India) + Stripe (Global)
Currency detection: IP geo + user setting. INR → Razorpay; everything else → Stripe.
Both providers' webhooks land at /api/webhooks/razorpay and /api/webhooks/stripe with signature verification.
Subscription lifecycle (created, renewed, payment_failed, cancelled) updates subscriptions and credit_ledger.
Dunning: 3 retries over 7 days, then downgrade to Free.
9. Free-Tier Keepalive Strategy
Supabase free tier pauses a project after 7 days of inactivity. Aether AI ships a built-in keepalive so this never happens.
9.1 The Heartbeat Pattern
A heartbeat table exists solely to be written to.
A /api/cron/heartbeat route runs an INSERT INTO heartbeat DEFAULT VALUES then DELETE FROM heartbeat WHERE beat_at < now() - interval '7 days'.
Both Vercel Cron and a backup scheduler (cron-job.org, UptimeRobot, or GitHub Actions schedule) hit the route every 6 hours.
A separate Supabase Edge Function (pg_cron-triggered every 24 hours) does an internal SELECT 1 too, so even if HTTP is blocked the DB itself stays warm.
// app/api/cron/heartbeat/route.ts
import { NextResponse } from "next/server";
import { createClient } from "@/lib/supabase/server";

export const runtime = "edge";
export const dynamic = "force-dynamic";

export async function GET(req: Request) {
  // Protect with a shared secret
  const auth = req.headers.get("x-cron-secret");
  if (auth !== process.env.CRON_SECRET) {
    return NextResponse.json({ ok: false }, { status: 401 });
  }

  const sb = createClient({ service: true });
  await sb.from("heartbeat").insert({});
  await sb.from("heartbeat")
    .delete()
    .lt("beat_at", new Date(Date.now() - 7 * 86_400_000).toISOString());

  return NextResponse.json({ ok: true, at: new Date().toISOString() });
}
# vercel.json
{
  "crons": [
    { "path": "/api/cron/heartbeat", "schedule": "0 */6 * * *" }
  ]
}
On Netlify, replace vercel.json with a Scheduled Function (netlify.toml schedule = "0 */6 * * *"). Add UptimeRobot pinging the same URL every 5 minutes as a free belt-and-braces.
9.2 Other Free-Tier Guardrails
Storage: lazy-load brand assets via Supabase Storage CDN; resize/convert to webp on upload.

11. Security, Privacy, Compliance
All API keys encrypted at rest with AES-256-GCM; encryption key in env, not DB.
Row-Level Security on every tenant table; service-role key never reaches browser.
HMAC signature verification on every inbound webhook (Stripe, Razorpay, n8n, WordPress).
Rate limiting on chat endpoint (per bot, per IP) via Upstash Redis + sliding window.
Content safety: per-bot toggle for moderation pre/post-filter using OpenAI Moderation or Llama Guard.
GDPR / DPDP ready: per-user data export and erasure; audit log retains hash-only after erase.
SOC2 path: structured logs, change management via PRs, secrets in vault, MFA on admin accounts.

14. Idea Improvements & Future Roadmap
14.1 Refinements I'd make to your spec
Universal skills marketplace: let any client publish their n8n skill as a public skill others can install in one click (revenue share).
White-label tier: agencies can resell Aether AI under their own brand with their own pricing — a real money-maker.
Bot 'memory' across conversations per end-user (opt-in) — turns the bot into a real assistant, not a goldfish.
Per-bot A/B testing: ship two system prompts or two themes and auto-pick the winner on a chosen metric (CSAT, conversion).
Voice channel: the same bot answers a phone number via Twilio + ElevenLabs/OpenAI Realtime. Huge for restaurants and clinics.
Outbound mode: scheduled bot-initiated messages on WhatsApp/email (n8n-triggered). Turns the chatbot into a CRM.
Tenant-level cost dashboard with predicted bill, so clients are never surprised.
Edge caching of identical prompts for 60s — saves real money at scale.
Plugin marketplace inside the dashboard, not just skills — themes, prompt packs, industry templates.
Compliance pack: India DPDP + EU GDPR + US CCPA toggles per tenant.
14.2 Future Tech Upgrades
Move heavy embeddings + rerank to a dedicated Modal/Replicate worker when free-tier limits start to bite.
Add Cloudflare in front for WAF + image resizing + R2 mirror of Supabase Storage.
Migrate hot tenants to dedicated Supabase projects via a routing table — same code, zero rewrite.
On-device inference for tiny tasks via WebGPU (intent classification before calling the cloud).
Native mobile (Expo) sharing 90% of the code via tRPC + Tamagui.
